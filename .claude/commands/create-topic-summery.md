## コマンドの目的

社内の Slack スレッドに投稿された技術記事のリンクやコメントを元に、トピックごとの要約を作成します。
作成された要約の例としては以下のファイルの「`## 🗓 Monthly Articles`」の項を参照してください。

- src/content/posts/2025/01.md
- src/content/posts/2025/06.md

## コマンドで行ってもらう操作

このコマンドでは以下の順番に従って操作を行なってください。

1. 指定された Slack スレッドの内容をまとめた Markdown ファイルを読み込みます。コマンド実行時に指定されなかった場合はユーザーの入力を求めてください。
2. 最終的に出力する Markdown ファイルのパスをユーザーに尋ねます。
3. 2 で尋ねたファイルがある確認します。ファイルが存在しない場合新しく作成します。そのファイルに後述する「トピックまとめのテンプレート」で指定したテンプレートを追加します。
4. 後述の「入力ファイルの形式と読み方」を元に、話題(h2 セクション)ごとに入力ファイルの記述を分割し、それぞれ TODO リストの項目に追加してください。
5. 3 で分割した話題ごとに、後述の「出力してほしい方式と変換方法」に従って変換してください。
6. 4 で変換した話題を後述の「トピックの分類方法」に従って分類し、3. で作成したファイルの適切な分類の見出しの下に追記してください。
7. ６の操作を 4 で分割した全ての話題に対して行い、都度 TODO リストの項目を完了していってください。
8. ここまでで、元々のファイルに存在していた話題が全て変換され、分類されていることを確認してください。
9. 最終的に作成されたファイルのパスをユーザーに伝えて終了です。

以降、これらの作業に必要な情報を記述します。

## トピックまとめのテンプレート

```md
### 📖 Framework, Library

### ⚡️ Services

### 🖥 Browsers

### 💬 Languages

### 🤖 Runtimes

### 📝 Specifications

### 🦆 Others
```

## 入力ファイルの形式と読み方

入力されるファイルは以下のような形式のデータが連続する Markdown ファイルです。

```md
## タイトル or 見出し

<!-- 以下は順不同 -->

• <URL>
• 説明
◦ ....
```

このような形式で一つの話題を表現しています。基本的に h2 で始まる見出しがタイトルとなりますが、内容にそこまで関係ない場合もあるので基本的には URL で書いてあるページのタイトルの方を優先します。

`◦`や`•` で始まる箇条書きで記事の URL や説明が書かれています。URL が一つの場合そのページが紹介している記事になります。複数ある場合はより内容の詳細度が高いものが優先されます。

## 出力してほしい方式と変換方法

まず一つの記事(話題)は以下のような一つのリスト要素に変換してください。

```md
- [title](blog/page url)
  - １文から２文の簡潔なまとめ
```

ただし必ず以下の点に注意して変換してください。

- URL は Markdown のリンク形式に変換すること。
- タイトルは基本的に URL の先のページのタイトル要素を最優先にすること
  - どうしてもその page でのタイトルが取得できない場合は、説明文を参考にしてタイトルを決めること。
- 入力の説明を全て完全に信用するのではなく、都度掲載されている URL のページを確認し、内容を把握してから変換すること。
- 明らかに技術記事やブログ記事などについてではないコメントは skip すること。
- 重複した記事は一つにまとめること。
- もし掲載されている URL が存在しない場合や、内容をうまく取得できない場合は、入力の説明を元に変換すること。
  - 説明すらも不足しており、想像でしか内容を把握できない場合は、説明部分に `TODO: 内容を確認して変換する`と書くこと。
- また、フォーマットとしては `src/content/posts/2025/01.md` ~ `src/content/posts/2025/08.md` も参考にすること。

## トピックの分類方法

変換の終了した topic は以下の 7 種類に分類されます。以下の基準をもとにどの分類に入るかを判断してください。

- 📖 Framework, Library : 特定のフレームワークやライブラリのアップデート情報や使い方などの記事を分類する。フロントエンド系のツールもここに分類する。
- ⚡️ Services : ホスティングサービスや API、AI サービスなど、フロントエンド開発に関連するサービスの記事を分類する。
- 🖥 Browsers : ブラウザの機能追加などについての情報を分類する
- 💬 Languages : CSS や JS、TS などのプログラミング言語に関する記事(使い方や TS のアップデート)を分類する
- 🤖 Runtimes : ブラウザの JS ランタイムや、Node.js,deno,Bun など JS ランタイムについての話題を分類する。
- 📝 Specifications :CSS や JS の特に仕様に関するアップデートや新しい Proposal についてを分類する。
- 🦆 Others : 上記の分類しづらかったものを分類する。
